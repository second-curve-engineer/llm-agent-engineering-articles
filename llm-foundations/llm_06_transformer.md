# Transformer 架构一图读懂：从 BERT 到 GPT 的分叉路

---

前几篇我们依次拆解了 Token、Embedding、Attention。现在把这些零件组装起来，看清楚 Transformer 的完整面貌。

先回顾一下这条链路：

> 文本 → Token → Embedding（向量） → Attention（上下文融合） → 输出

Transformer 就是把这个过程做得极深、极宽的神经网络：多层 Attention 堆叠，每一层都让向量表示变得更丰富，最终输出一个对语言结构理解深刻的表示。

但 Transformer 不是一种架构，而是一个家族。BERT 和 GPT 都是 Transformer，却走上了完全不同的路。理解这个分叉，是理解当前 LLM 格局的基础。

---

## Transformer 的基本单元：一层在做什么

Transformer 的每一层，结构都相同，由两个子模块串联组成：

**第一个子模块：Multi-Head Attention**

把上一层输出的向量序列作为输入，让每个 Token 的向量通过 Attention 机制融合序列中其他 Token 的信息，输出一组新的向量。

这一步解决的是"关系"问题：词与词之间的依赖，谁修饰谁，谁指代谁。

**第二个子模块：Feed-Forward Network（FFN）**

把 Attention 输出的每个向量，独立地通过一个小型全连接网络做变换，输出新的向量。

这一步解决的是"知识"问题：在向量空间里对信息做进一步的非线性变换，让模型有能力存储和激活更复杂的知识模式。

两个子模块之后，都会做 **残差连接（Residual Connection）** 和 **层归一化（Layer Norm）**——残差连接让信息可以"绕过"当前层直接传递，避免深层网络的梯度消失；层归一化让训练更稳定。

这是一层的完整结构。GPT-4 有 128 层，这个结构重复了 128 次。

---

## 三种架构：Encoder、Decoder、Encoder-Decoder

Transformer 家族根据用途，演化出了三种主要架构形态。

### Encoder（编码器）：BERT 为代表

Encoder 的核心特征：**双向 Attention**——每个 Token 可以同时看到它左边和右边的所有词。

输入一段文本，Encoder 输出的是每个 Token 的向量表示，这个表示融合了整句话的上下文信息。

这种设计天然适合**理解任务**：
- 文本分类（这段话是正面还是负面情绪？）
- 命名实体识别（哪些词是人名？哪些是地名？）
- 语义匹配（这两个句子意思一样吗？）
- Embedding 生成（把文本压缩成一个向量）

BERT 的预训练任务是**Masked Language Model（MLM）**：随机遮住输入中的一些词，让模型预测被遮住的词是什么。因为需要根据上下文（包括右边的词）来猜，所以必须是双向的。

**BERT 系列的代表模型**：BERT、RoBERTa、DeBERTa、ALBERT

### Decoder（解码器）：GPT 为代表

Decoder 的核心特征：**单向 Attention（Causal Attention）**——每个 Token 只能看到它左边的词，右边的被遮住。

输入一段前缀文本，Decoder 的任务是预测下一个词，然后把预测出的词拼到序列末尾，再预测下一个，循环往复，直到生成结束符号。

这种设计天然适合**生成任务**：
- 对话（根据对话历史生成下一轮回复）
- 写作（续写文章、生成代码）
- 推理（一步一步推导出结论）

GPT 的预训练任务是 **Next Token Prediction（NTP）**：给定前面的词，预测下一个词。因为不能"偷看"后面的词，所以必须是单向的。

**GPT 系列的代表模型**：GPT-3/4、Claude、Llama、Mistral、Qwen、DeepSeek

### Encoder-Decoder：T5、BART 为代表

把 Encoder 和 Decoder 组合起来，适合**输入和输出都是序列，且两者之间需要对齐**的任务：
- 机器翻译（中文 → 英文）
- 摘要生成（长文 → 短摘要）
- 问答（文档 + 问题 → 答案）

Encoder 先"读懂"输入，产生一组表示；Decoder 再根据这组表示，逐词生成输出。Decoder 里有一种特殊的 **Cross-Attention**：Query 来自 Decoder，Key 和 Value 来自 Encoder 的输出，让 Decoder 在生成每个词时能参考输入的全部信息。

**代表模型**：T5、BART、mT5

## 分叉的根本原因：预训练目标不同

很多人以为 BERT 和 GPT 的差别是"双向 vs 单向"，但这只是表象。

**根本原因是预训练目标不同，预训练目标决定了架构选择。**

- BERT 的目标：预测被遮住的词 → 需要看两侧的上下文 → 必须双向 → Encoder
- GPT 的目标：预测下一个词 → 只能看左侧的历史 → 必须单向 → Decoder

不是"设计者觉得双向好所以用了 Encoder"，而是"想做 MLM 训练，所以必须用 Encoder"。架构服从于任务。

这个逻辑也解释了为什么当前主流 LLM 几乎都是 Decoder 架构：

Next Token Prediction 是最简单、最通用、最容易扩展到海量数据的训练目标。不需要特殊的标注，任何文本都可以直接用来训练。随着模型规模增大，这个看似简单的任务，催生出了涌现的推理能力。

而 MLM 需要人为设计遮掩策略，且 Encoder 的输出是表示向量，不能直接用来生成文本——这限制了它在生成和推理场景下的应用。

## 层数、参数量、涌现能力

一个常见的问题：为什么 GPT-4 比 GPT-3 强那么多？

不只是参数量更大，更重要的是**涌现（Emergence）**。

研究发现，当模型的参数量超过某个阈值时，某些能力会突然出现——在小模型上完全看不到，在大模型上突然就会了。比如：
- 少样本学习（给几个例子就能举一反三）
- 链式推理（step by step 思考）
- 指令遵循（按照复杂指令完成任务）

这些能力不是被人工设计进去的，而是从"预测下一个词"这个任务里，随着规模增大自然涌现出来的。

这也是为什么 Scaling Law（规模定律）是当前 LLM 研究的核心：模型越大、数据越多、计算越多，能力就越强，而且这个关系是可以预测的。

## 对 Agent 设计的实际影响

**1. 选对模型类型**

做 RAG 的 Embedding 生成，选 Encoder 类模型（如 BGE、text-embedding 系列）。做对话、推理、工具调用，选 Decoder 类模型（GPT、Claude、Llama 等）。两者不要混用。

**2. 理解"层数"对能力的影响**

小模型（7B 参数以下）适合简单的单步任务：分类、提取、格式转换。复杂的多步推理、工具调用链、需要跨轮记忆的任务，需要更大的模型。不要用小模型做超出其能力边界的事，结果不稳定，调 prompt 也救不回来。

**3. 同一家族的模型，行为模式更一致**

Llama 3 和 Llama 2 的提示词格式、行为模式有更多共性，比 Llama 和 GPT 之间的差异小得多。在 Agent 里切换模型时，优先在同一家族内切换，迁移成本更低。

**4. Decoder 的生成是不可逆的**

Decoder 逐词生成，一旦生成了某个词，后续的生成都建立在这个词之上。这就是为什么 LLM 会"一错再错"——一旦走上错误路径，很难自我纠正。在 Agent 设计里，对关键决策节点加验证和重试机制，比期望模型自我纠错更可靠。

---

## 小结

Transformer 家族的三种形态，本质上是三种不同的预训练目标催生出的三种架构：

- **Encoder（BERT 类）**：双向 Attention，适合理解和表示任务
- **Decoder（GPT 类）**：单向 Attention，适合生成和推理任务
- **Encoder-Decoder（T5 类）**：组合两者，适合序列到序列的转换任务

当前主流 LLM 几乎全是 Decoder 架构，原因不是 Decoder"更好"，而是 Next Token Prediction 这个训练目标在规模扩展上有巨大优势，催生出了涌现的推理能力。

理解这个分叉，你才能看懂 LLM 发展的脉络，也才能在技术选型时做出有根据的判断。
